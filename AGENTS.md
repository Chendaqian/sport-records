## Project Overview

Fork of [yihong0618/running_page](https://github.com/yihong0618/running_page) — a personal sports activity visualization page. Syncs workout data from fitness platforms (Garmin/Strava/Nike/Keep/Coros/etc.), stores in SQLite, generates SVG visualizations, and displays an interactive map frontend.

## Common Commands

### Frontend (Node/pnpm)

```bash
pnpm develop          # Dev server (Vite)
pnpm build            # Production build -> ./dist
pnpm lint             # ESLint with auto-fix on src/
pnpm check            # Prettier format check
pnpm ci               # check + lint + build (CI pipeline)
```

### Python Data Sync

```bash
pip install -r requirements.txt

# Sync from platforms (requires secrets/tokens)
python run_page/garmin_sync.py <secret_string>
python run_page/garmin_sync.py <secret_string> --is-cn          # Garmin CN
python run_page/strava_sync.py <client_id> <client_secret> <refresh_token>
python run_page/nike_sync.py <nike_refresh_token>
python run_page/keep_sync.py <phone> <password>

# Generate SVG visualizations
python run_page/gen_svg.py --from-db --type github --output assets/github.svg --use-localtime
python run_page/gen_svg.py --from-db --type grid --output assets/grid.svg --use-localtime
python run_page/gen_svg.py --from-db --type circular --use-localtime
```

### Python Linting

```bash
pip install -r requirements-dev.txt
black . --check       # Format check (used in CI across Python 3.9-3.12)
```

### Clean Data

```bash
pnpm data:clean       # Removes data.db, GPX/TCX/FIT outputs, activities, activities.json
```

## Architecture

### Backend (Python — `run_page/`)

**Data flow: Platform API -> Sync Script -> SQLite (`run_page/data.db`) -> `src/static/activities.json`**

- **Sync scripts** (`*_sync.py`): Each fitness platform has a dedicated sync script. They pull activity data from the platform API and write to SQLite via the Generator/DB layer.
- **`run_page/generator/db.py`**: SQLAlchemy ORM layer. `Activity` model is the core table with fields: `run_id`, `name`, `distance`, `moving_time`, `type`, `start_date_local`, `location_country`, `summary_polyline`, `average_heartrate`, `average_speed`.
- **`run_page/generator/__init__.py`** (`Generator` class): Orchestrates sync — reads from data directories or Strava API, upserts into SQLite, exports to JSON.
- **`run_page/config.py`**: Defines file paths (`SQL_FILE`, `JSON_FILE`, `GPX_FOLDER`, etc.), reads optional `config.yaml` for credentials.
- **`run_page/utils.py`**: Shared helpers — timezone adjustment, Strava client creation, `make_activities_file()` which is the main entry to go from raw files -> DB -> JSON.
- **`run_page/gen_svg.py`**: Generates GitHub-style heatmap, grid, and circular SVG posters from the SQLite database.
- **`run_page/gpxtrackposter/`**: SVG drawing library used by `gen_svg.py`.
- **`run_page/polyline_processor.py`**: Filters/adjusts polyline coordinates (privacy zones via `IGNORE_POLYLINE` env var).

### Frontend (React/TypeScript — `src/`)

**Stack**: React 18, TypeScript, Vite, Tailwind CSS, Mapbox GL, react-map-gl

- **`src/pages/index.tsx`**: Main page — renders map, year stats, location stats, run table. All state management is local (useState).
- **`src/utils/utils.ts`**: Core utility — `Activity` interface, polyline decoding, GeoJSON generation, filtering/sorting, pace/time formatting, coordinate transforms (GCJ02/WGS84 via gcoord).
- **`src/utils/const.ts`**: All configuration constants — Mapbox token, color scheme per activity type, Chinese/English i18n strings, map styling options.
- **`src/static/activities.json`**: Generated by Python backend — the data source for the frontend. Imported at build time.
- **`src/hooks/useActivities.ts`**: Loads `activities.json` and computes year stats.
- **`src/components/RunMap/`**: Mapbox map with activity route polylines, markers, and controls.
- **`src/components/RunTable/`**: Sortable table of individual activities.
- **`src/components/LocationStat/`**: City/province/country statistics from reverse geocoding.
- **`src/components/YearsStat/`**: Year-by-year summary statistics.
- **`src/components/SVGStat/`**: Embeds the generated SVG posters.
- **`src/static/city.ts`**, **`src/static/run_countries.ts`**: Chinese city/region lookup data for location parsing.

Path aliases: `@/*` -> `./src/*`, `@assets/*` -> `./assets/*`

### CI/CD (`.github/workflows/`)

- **`run_data_sync.yml`**: Runs daily (cron) or on push to master. Syncs data from configured platform, generates SVGs, commits results. Configured via env vars at top of file (`RUN_TYPE`, `ATHLETE`, `TITLE`, etc.).
- **`gh-pages.yml`**: Builds frontend with pnpm and deploys to GitHub Pages.
- **`ci.yml`**: Python lint check (`black`) across Python 3.9-3.12.

### Deployment

- **GitHub Pages**: Via `gh-pages.yml` workflow, `PATH_PREFIX` set to repo name.
- **Vercel**: `vercel.json` disables gh-pages branch deployment. Uses default Vite output.
- **Docker**: Multi-stage Dockerfile — Python sync stage -> Node build stage -> nginx serve.

### Key Data Files (generated, not hand-edited)

| File                               | Purpose                                                |
| ---------------------------------- | ------------------------------------------------------ |
| `run_page/data.db`                 | SQLite database of all activities                      |
| `src/static/activities.json`       | JSON export consumed by frontend                       |
| `assets/github.svg`                | GitHub-style heatmap poster                            |
| `assets/grid.svg`                  | Grid poster                                            |
| `GPX_OUT/`, `TCX_OUT/`, `FIT_OUT/` | Downloaded activity files                              |

### Activity Types Supported

Run, Trail Running, Ride, Indoor Ride, VirtualRide, Hike, Swim, Rowing, Kayaking, Snowboard, Ski, RoadTrip, Flight — each mapped to a color in `const.ts`.
